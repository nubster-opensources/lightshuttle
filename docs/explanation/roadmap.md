# LightShuttle roadmap

The roadmap is derived from GitHub milestones, which are the single source of truth : no hard release dates are committed. LightShuttle is a developer-time orchestrator ; production runtimes, a Kubernetes replacement, a service mesh and CI/CD pipelines are permanently out of scope.

| Version | Theme | Status |
|---------|-------|--------|
| [v0.1.0](https://github.com/nubster-opensources/lightshuttle/milestones) | Minimum viable orchestrator | Shipped |
| [v0.2.0](https://github.com/nubster-opensources/lightshuttle/milestones) | Dashboard and observability | Shipped |
| [v0.3.0](https://github.com/nubster-opensources/lightshuttle/milestones) | Production export | Shipped |
| [v0.4.0](https://github.com/nubster-opensources/lightshuttle/milestones) | Secrets and env management | Shipped |
| [v0.5.0](https://github.com/nubster-opensources/lightshuttle/milestones) | Polish and stability | Shipped |
| [v0.6.0](https://github.com/nubster-opensources/lightshuttle/milestones) | Polyglot resources | Planned |
| [v0.7.0](https://github.com/nubster-opensources/lightshuttle/milestones) | Extension and lifecycle | Planned |
| [v1.0.0](https://github.com/nubster-opensources/lightshuttle/milestones) | Stable | Planned |

Milestones follow the naming format `vX.Y.0 - Theme` (example: `v0.6.0 - Polyglot resources`), include a short description, and list the issues attached to that scope. Full design notes for a given release live under `docs/design/` and `docs/spec/`.
